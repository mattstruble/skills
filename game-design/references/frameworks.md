## MDA Framework

**Source:** Hunicke, LeBlanc, Zubek (2004). "MDA: A Formal Approach to Game Design and Game Research."

The pipeline: **Mechanics → Dynamics → Aesthetics**

- **Mechanics**: The rules, data, and algorithms the designer writes. Everything the designer directly controls.
- **Dynamics**: Runtime behavior that emerges when mechanics interact with player input and each other. The designer cannot directly write dynamics — only create conditions for them.
- **Aesthetics**: The emotional responses the player experiences. What the game *feels like* to play.

**The key insight**: Designers see mechanics first and aesthetics last. Players see aesthetics first and mechanics last. This asymmetry is why playtesting is irreplaceable — the designer's perspective is structurally inverted from the player's.

### LeBlanc's 8 Kinds of Fun

MDA's taxonomy of aesthetic experiences:

| Kind | Description | Example |
|---|---|---|
| **Sensation** | Game as sense-pleasure | Audiovisual spectacle, haptic feedback |
| **Fantasy** | Game as make-believe | Playing a wizard, inhabiting a world |
| **Narrative** | Game as drama | Story-driven games, emergent stories |
| **Challenge** | Game as obstacle course | Competitive games, hard platformers |
| **Fellowship** | Game as social framework | Co-op, MMOs, party games |
| **Discovery** | Game as uncharted territory | Exploration, hidden mechanics |
| **Expression** | Game as self-discovery | Character builders, creative sandboxes |
| **Submission** | Game as pastime | Idle games, casual loops |

Most successful games serve 2-3 aesthetics simultaneously. A game that serves only one tends to feel one-note.

### MDA Strengths and Weaknesses

**Strengths**: Formalized the layered relationship between design and experience. Introduced iterative design thinking — evaluate at the aesthetics layer, intervene at the mechanics layer. Gave designers a shared vocabulary.

**Weaknesses**:
- "Mechanics" conflates everything the designer controls — code architecture, visual style, sound design, world concept. This is too broad to be actionable.
- Narrative is treated as one aesthetic among eight, not as something woven through all layers. This leads to narrative being bolted on rather than integrated.
- "Aesthetics" is philosophically overloaded — it means both "the sensory layer" (common usage) and "the emotional experience" (MDA's usage). Causes confusion in discussion.

The DDE framework addresses all three weaknesses.

## DDE Framework

**Source:** Walk, Görlich, Barrett (2017). "Design, Dynamics, Experience."

The pipeline: **Design → Dynamics → Experience**

Design is subdivided into three components:

- **Blueprint**: World concept, style guides, rule documentation, the game's "why." Everything that exists before a line of code is written.
- **Mechanics** (narrowed from MDA): Code architecture, game rules at the implementation level. What the engine actually executes.
- **Interface**: Everything the player perceives — graphics, sound, UI, haptics, feedback. The sensory surface of the game.

Experience is subdivided into three journeys:

- **Sensory Journey**: What the player perceives through the interface — visuals, audio, tactile response.
- **Emotional Journey** (Cerebellar): Feelings generated by the dynamics — excitement, frustration, triumph, dread.
- **Intellectual Journey** (Cerebral): Meaning-making, pattern recognition, strategic reasoning.

### The Player-Subject

From Sicart (2009), integrated into DDE: the **Player-Subject** is the mental persona that plays the game — not the full human, but a subset with different abilities, ethics, and risk tolerance. The Player-Subject can make decisions the real person never would. Designers target the Player-Subject, not the player directly.

### The Antagonist

DDE introduces the **Antagonist** as the game itself conceived as a unified challenge-entity — the sum of enemy AI, level design, physics, time pressure, and resource scarcity. The Antagonist is not a character; it's the game's resistance to the player's goals. Ludo-narrative dissonance occurs when the Antagonist is inconsistent — when the game's rules say one thing and the story says another.

### DDE Strengths Over MDA

- **Narrative is structural, not aesthetic**: Blueprint includes the world concept and story intent; Interface includes the narrative presentation; Dynamics include emergent story. Narrative runs through all three layers rather than being one item in a list.
- **Ludo-narrative dissonance gets a structural explanation**: When Blueprint (intended story) conflicts with Dynamics (emergent behavior), the Antagonist becomes incoherent. The framework names the problem precisely.
- **Interface is separated from Mechanics**: A game's visual style is not the same kind of thing as its physics engine. DDE treats them separately, which is more actionable for diagnosis.

## Schell's Elemental Tetrad

**Source:** Jesse Schell (2008). *The Art of Game Design: A Book of Lenses.*

Four elements, all directly connected to each other:

```
      Mechanics
     /          \
Aesthetics ---- Story
     \          /
      Technology
```

- **Mechanics**: Rules, goals, procedures, conflict. What the player does.
- **Aesthetics**: Sensory experience — visuals, sound, feel. What the player perceives.
- **Story**: Sequence of events — embedded narrative, characters, world. What the player understands.
- **Technology**: The medium — digital engine, physical components, paper. What makes the game possible.

**The key insight**: "The game enables the experience, but it IS NOT the experience." Schell uses this to argue that all four elements must serve the experience — none is more fundamental than another. The elements are interdependent: privileging any one (including Mechanics) leads to incomplete design.

**The Lenses approach**: Schell's book provides 100+ "lenses" — questions to evaluate design from different perspectives. Examples: "Lens of the Toy" (is this fun to play with even without rules?), "Lens of Surprise" (what will surprise the player?), "Lens of Fairness" (is the game fair from all players' perspectives?).

### Tetrad vs MDA

The Tetrad does not use layered abstraction — all four elements connect directly to each other. This is more honest about how games actually work (technology directly affects aesthetics; story directly affects mechanics) but loses MDA's insight about the designer/player perspective asymmetry.

The Tetrad explicitly includes Technology, which MDA ignores. This matters: the stamina system in Dark Souls is a technological choice (how to implement resource depletion) that directly creates mechanical depth. The Tetrad can name this relationship; MDA cannot.

## Costikyan's Critical Vocabulary

**Source:** Greg Costikyan (1994, revised 2002). "I Have No Words & I Must Design." Originally published in *Interactive Fantasy*.

**Definition**: Games are interactive structures of endogenous meaning requiring struggle toward a goal.

Key terms:
- **Interaction**: The player must make decisions that matter. A game without meaningful decisions is not a game.
- **Goals**: Players must have something to strive for. Without goals, there is no struggle.
- **Struggle**: Overcoming obstacles is the core activity. Trivial obstacles produce trivial games.
- **Endogenous Meaning**: Objects and actions acquire meaning within the game's context. A chess queen has no meaning outside chess. This is what separates games from simulations.

### Costikyan's Distinctions

| Form | Difference from Games |
|---|---|
| **Puzzles** | Lack dynamic interaction — the puzzle is fixed, the solution is fixed. Once solved, a puzzle is exhausted. |
| **Stories** | Immutable sequences — the reader/viewer cannot change outcomes. Interactivity is illusory. |
| **Toys** | No fixed rules or goals — play is open-ended. Toys become games when goals are added. |

These distinctions are useful for diagnosing hybrid forms. A "walking simulator" is closer to a story than a game by Costikyan's definition. That's not a criticism — it's a precise description that clarifies what the designer is actually making.

## Lazzaro's 4 Keys to More Emotion

**Source:** Nicole Lazzaro (XEODesign, 2004). "Why We Play Games: Four Keys to More Emotion Without Story."

Four distinct emotional experiences players seek:

| Key | Core Emotion | Mechanism | Example |
|---|---|---|---|
| **Hard Fun** | Fiero (triumph over adversity) | Meaningful challenges, strategies, puzzles | Dark Souls, Tetris, competitive FPS |
| **Easy Fun** | Wonder, awe, mystery | Ambiguity, incompleteness, rich detail | Minecraft exploration, open-world discovery |
| **People Fun** | Social emotions (schadenfreude, naches) | Competition, cooperation, performance, spectacle | Mario Kart, Among Us, MMO raiding |
| **Serious Fun** | Meaning, value, purpose | Clearing the mind, feeling better, real-world impact | Puzzle games as meditation, educational games |

**Fiero** deserves special attention: it's the emotion of triumph over a worthy adversary. It's the fist-pump moment. Hard Fun games are engines for generating fiero — they create frustration deliberately so that eventual success produces a proportionally intense emotional release.

**The key insight**: Most successful games serve multiple keys. A game that serves only Hard Fun will lose players who don't experience fiero as rewarding. Supplementing one key with elements of another enriches the experience without diluting it. Dark Souls is primarily Hard Fun but has Easy Fun (world exploration, lore discovery) and People Fun (invasions, jolly cooperation) woven through it.

For full treatment of motivation models including how 4 Keys relates to LeBlanc's 8 Kinds of Fun, see the "Player Motivation Models" section in `player-experience.md` (if not already loaded).

## Crawford's Definitions

**Source:** Chris Crawford (1984). *The Art of Computer Game Design.*

Crawford's definition predates MDA but remains useful for its precision:

Games are **closed formal systems that subjectively represent a subset of reality.**

Three requirements:
1. **Representation**: The game models some aspect of reality (abstract or concrete).
2. **Interaction**: The player can affect the model.
3. **Conflict**: The player's goals are opposed by something — another player, the environment, time, the player's own limitations.

Crawford's distinctions (similar to Costikyan's but earlier):
- **Puzzles**: Lack dynamic interaction. The puzzle doesn't respond to the player's approach.
- **Stories**: Lack meaningful interaction. The player observes but cannot change outcomes.
- **Toys**: Lack conflict and goals. Play is open-ended.

Crawford's framework is more restrictive than Costikyan's — it excludes cooperative games without opposition and games where conflict is entirely internal. Use it as a starting point, not a final definition.

## Church's Formal Abstract Design Tools

**Source:** Doug Church (1999). "Formal Abstract Design Tools."

Church's project: extract reusable design concepts from existing games, rather than inventing new vocabulary from scratch.

Two key concepts:

**Intention**: The player makes an implementable plan based on understanding their options. For intention to exist, the player must be able to:
1. Perceive the current game state clearly
2. Understand what actions are available
3. Predict the consequences of those actions
4. Form a plan they believe will work

Games that obscure any of these four undermine intention. Players who cannot form intentions are not playing — they're guessing.

**Perceivable Consequences**: The game world reacts clearly to player actions. The player can see what their choice caused. Without perceivable consequences, there is no feedback loop, and therefore no learning.

Church's framework is less a complete theory and more a diagnostic tool. When a game feels opaque or frustrating, check intention and perceivable consequences first.

## Worked Example: Dark Souls Through Three Lenses

**MDA Analysis:**

- *Mechanics*: Stamina bar, hitboxes, iframes on dodge, enemy attack telegraphs, estus flask charges, bonfire checkpoints.
- *Dynamics*: Combat becomes a resource management problem. Stamina creates tradeoffs between offense and defense. Iframes reward precise timing. Bonfires create tension arcs — the further from a bonfire, the higher the stakes.
- *Aesthetics*: Challenge (primary), Discovery (lore, hidden paths), Fantasy (inhabiting a dying world).

MDA sees the combat mechanics generating punishing dynamics creating aesthetic satisfaction. It cannot explain *why* the world design reinforces the combat feel — that's outside its vocabulary.

**DDE Analysis:**

- *Blueprint*: A dying world where the player is already dead. The world concept is one of entropy and persistence. This informs every design decision.
- *Mechanics*: Same as MDA's mechanics layer.
- *Interface*: Enemy design communicates threat level visually. Sound design (footsteps, weapon impacts, ambient dread) is part of the interface, not the mechanics. The consistent visual language of "dangerous things look dangerous" is an interface decision.
- *Dynamics*: Same as MDA.
- *Experience*: The Intellectual Journey (pattern recognition, learning enemy tells) and Emotional Journey (frustration → fiero) are both served. The Sensory Journey (oppressive atmosphere) reinforces both.

DDE additionally sees the world design (Blueprint), the consistent rules as Interface, and the Player-Subject's intellectual/emotional journey as distinct layers. The Antagonist (the game's unified resistance) is coherent — the world, enemies, and rules all say the same thing: *you are not welcome here, but you can learn.*

**Schell's Tetrad Analysis:**

- *Mechanics*: Stamina, iframes, estus, bonfires.
- *Aesthetics*: Gothic architecture, mournful sound design, deliberate animation weight.
- *Story*: The dying world, the cycle of fire, the player's role as undead.
- *Technology*: The stamina system and hitbox precision are technological choices that directly create mechanical depth. The "weight" of animations is a technological choice that creates aesthetic satisfaction.

The Tetrad highlights what MDA misses: the technology (stamina system, hitbox precision) directly connects to story (the dying world mirrors the player's struggle — everything costs something, nothing is free). The Tetrad can name this four-way connection; MDA and DDE cannot as directly.

**What each framework reveals:**
- MDA: The causal chain from rules to experience. Useful for diagnosing why a mechanic isn't producing the intended feel.
- DDE: Why narrative and gameplay feel unified. Useful for diagnosing ludo-narrative dissonance.
- Tetrad: How technology choices create story meaning. Useful for justifying technical decisions in design terms.

## Choosing a Framework

No single framework is complete. Use them as lenses, not as competing theories:

| Use Case | Best Framework |
|---|---|
| Diagnosing why a mechanic doesn't feel right | MDA (trace mechanics → dynamics → aesthetics) |
| Diagnosing ludo-narrative dissonance | DDE (check Blueprint vs Dynamics coherence) |
| Justifying a technology choice in design terms | Schell's Tetrad |
| Defining what kind of thing you're making | Costikyan or Crawford |
| Diagnosing why a game feels one-note | Lazzaro's 4 Keys |
| Diagnosing why a game feels opaque | Church's Intention + Perceivable Consequences |
